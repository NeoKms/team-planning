# Project rules

## Общие правила

- Общение с пользователем — на русском.
- Перед изменениями смотри существующие файлы и паттерны проекта.
- Предпочитай минимальные изменения без лишних абстракций.
- Не сохраняй секреты и приватные данные в репозиторную память.

## Frontend

- Используй Vue 3 Composition API, если в соседнем коде нет другого явного паттерна.
- Для состояния используй Pinia, когда состояние должно быть общим для нескольких компонентов.
- Для маршрутизации используй существующий Vue Router.
- Для верстки и визуального дизайна используй Tailwind CSS utility-классы.
- UI-решения согласовывай с уже существующим стилем проекта.
- Не используй системные `window.confirm`/`alert` для пользовательских подтверждений; используй общий Vue confirm-dialog через `useConfirmDialog`.

## Проверки

- После изменений в TypeScript/Vue-коде по возможности запускай `npm run build`.
- После изменений, связанных с lint-правилами или форматированием, по возможности запускай `npm run lint`.
